# FMD Flashcard – Issue Pack (DE)

Dieser Ordner enthält kleine, gut strukturierte GitHub-Issues für die nächsten Entwicklungsschritte.

## Verwendung
1. Entsprechende GitHub-Issues im Repository anlegen.
2. Jede Markdown-Datei als Issue-Text kopieren.
3. Titel beibehalten; Scope bei Bedarf anpassen.

## Reihenfolge
Die Nummerierung schlägt eine sinnvolle Reihenfolge vor, kann aber angepasst werden:
- Erst Cards extrahieren (005–007)
- Dann Scheduling + Persistenz (008–009)
- UX-Verbesserungen (003–004, 011)
- Tooling/Watcher optional (010, 012)
