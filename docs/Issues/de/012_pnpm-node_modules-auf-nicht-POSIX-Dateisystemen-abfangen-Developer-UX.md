# pnpm/node_modules auf nicht-POSIX-Dateisystemen abfangen (Developer UX)

## Kontext
pnpm nutzt Symlinks/Hardlinks für `node_modules`. Auf exFAT/FAT/limitierten Mounts führt das zu EPERM-Fehlern.

## Ziel
Developer Experience verbessern, indem unsichere Dateisysteme erkannt werden und eine klare, umsetzbare Meldung erscheint.

## Aufgaben
- [ ] In `tools/control.py` bzw. Runner unter Linux FS-Typ für Repo-Pfad erkennen.
- [ ] Bei exFAT/FAT/unsupported für Symlinks warnen und vorschlagen:
  - Repo nach ext4/btrfs verschieben
  - oder `node_modules` per Bind-Mount auf Linux-FS auslagern
- [ ] Sicherstellen: `--start` überspringt Install nicht nur weil `node_modules` existiert; prüfen, ob `node_modules/.bin/tauri` existiert.

## Akzeptanzkriterien
- Auf unsupported FS erscheint eine klare Warnung bevor pnpm scheitert.
- Wenn `node_modules` unvollständig ist, führt das Tool `pnpm install` erneut aus.

## Hinweise
Dieses Issue betrifft nur Dev-Tooling, nicht den Enduser-Runtime-Pfad.
